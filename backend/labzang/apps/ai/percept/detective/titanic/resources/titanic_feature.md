# Titanic 데이터·피처 개요

## Overview

데이터는 두 그룹으로 나뉩니다.

- **training set** (`train.csv`)
- **test set** (`test.csv`)

**Training set**은 머신러닝 모델을 만드는 데 사용합니다. 학습용 데이터에는 각 승객의 **결과(ground truth)** 가 포함되어 있습니다. 모델은 승객의 성별·객실 등급 같은 **피처(feature)** 를 바탕으로 학습하며, 필요하면 피처 엔지니어링으로 새 피처를 만들 수 있습니다.

**Test set**은 모델이 **보지 못한 데이터**에서 얼마나 잘 동작하는지 확인하는 데 사용합니다. 테스트 세트에는 각 승객의 정답(생존 여부)이 주어지지 않으며, 학습한 모델로 타이타닉 침몰 시 **생존 여부를 예측**하는 것이 과제입니다.

또한 `gender_submission.csv`는 **모든 여성만 생존**한다고 가정한 예측 예시로, 제출 파일 형식 참고용으로 제공됩니다.

## Variable notes (원본 컬럼 설명)

### `pclass`

사회·경제적 지위(SES)의 대리 변수.

- **1st** — 상류
- **2nd** — 중류
- **3rd** — 하류

### `age`

- 1세 미만은 소수로 표현될 수 있음.
- 추정 나이는 `xx.5` 형태로 표기되는 경우가 있음.

### `sibsp`

가족 관계 정의:

- **Sibling** — 형제, 자매, 의붓형제, 의붓자매
- **Spouse** — 남편, 아내 (첩·약혼자 등은 제외)

### `parch`

가족 관계 정의:

- **Parent** — 어머니, 아버지
- **Child** — 딸, 아들, 의붓딸, 의붓아들

일부 아이는 유모만 동행한 경우 `parch = 0`일 수 있음.

## 컬럼·라벨 정리

> **컬럼(column) = 변수(variable) = 피처(feature)** (문맥에 따라 같은 의미로 쓰일 수 있음)

| 구분 | 이름 |
|------|------|
| 라벨(타깃) | `Survived` |
| 피처 | `Pclass`, `Name`, `Sex`, `Age`, `SibSp`, `Parch`, `Ticket`, `Fare`, `Cabin`, `Embarked` |

## Categorical (범주형)

데이터가 **카테고리**로 묶일 때 사용하는 척도입니다.

### Nominal (명명척도)

이름에 의한 구분. **순서는 의미 없이** 구분만 가능한 데이터.

- 예: 청팀, 홍팀, 백팀

### Ordinal (서열척도)

**순서(서열)** 가 있는 데이터.

- 예: 승리 가능성 — 1. 매우 낮음 → 2. 낮음 → 3. 보통 → 4. 높음 → 5. 매우 높음

## Quantitative (양적)

**숫자로 셀 수 있을 때** 사용하는 척도입니다.

### Interval (간격척도)

**절대적 영점이 없고**, 일정한 간격으로 측정되는 데이터. “10배 덥다”, “10배 시다”처럼 비율 해석은 부적절할 수 있음.

- 예: 시각 구간(11:00~11:05), 온도, pH

### Ratio (비율척도)

**의미 있는 영점(0)** 이 있어 비율 비교가 가능한 데이터.

- 예: 나이, 돈, 몸무게 (“10배 많다”와 같은 비율 해석이 가능)
